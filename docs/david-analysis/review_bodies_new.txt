# Missing from issue #3654                                                                                                  
                                                                   
  1. **`walletName` parameter threading through the translation chain** — `translate()`, `translateParams()`, `getMailAffix()
  `, `mapMailAffix()` all got `walletName` added as a parameter that's passed down 4 levels deep. This should be refactored i
  nto a translation context/strategy object that's resolved once at the top of `createUserV2Mail`, rather than threaded throu
  gh every method call. F1 partially addresses the notification service side, but the MailFactory internal plumbing is still 
  messy.                                                                       
                                                                               
  2. **The `.filter((a) => a.text || a.url || a.mail)` added to `getMailAffix`** — this filters out empty lines (`DefaultEmpt
  yLine` has `text: ''`), which changes spacing behavior for ALL wallets, not just RealUnit. That's a regression risk for exi
  sting DFX mails.
